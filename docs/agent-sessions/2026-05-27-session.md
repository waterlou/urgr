# Session 2026-05-27

## Goal
Fix various bugs found while testing scan/build functionality on FBNeo collection.

## Changes

### Bug fixes

1. **Tracing subscriber polluting stdout** (`apps/build-cli/src/main.rs`, `apps/scraper-cli/src/main.rs`)
   - Added `.with_writer(std::io::stderr)` to `tracing_subscriber::fmt()` so tracing `info!()` output goes to stderr, not stdout
   - Fixes JSON parsing in `execCliStream` which does `JSON.parse(stdout.trim())`

2. **Prior versions including newer versions** (`libs/rom-manager/src/builder.rs`)
   - Filter `prior_versions` to only include versions older than the current one using `version_cmp(l, &latest.version) == Less`
   - Fixes scan/build showing wrong "reused" count (e.g., counting nightly's files as reusable when building v1.0.0.02)

3. **Folder/slug separation** (`apps/rom-manager-ui/src/components/CollectionForm.jsx`)
   - Added separate `folder` state variable and editable field in the form
   - No longer forces `folder = slug` on save

4. **Rename logic using slug instead of folder** (`apps/rom-manager-ui/server/routes/collections.js`)
   - Fixed to use `col.folder` (actual data directory) instead of `col.slug` when determining old directory for rename

5. **IA.js missing slug fallback** (`apps/rom-manager-ui/server/routes/ia.js`)
   - Added slug as fallback when folder is null
