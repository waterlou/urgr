## API Endpoints
| Endpoint | Method | Purpose |
|---|---|---|
| `/api/downloads/enqueue` | POST | Add game's files to queue |
| `/api/downloads` | GET | List queue items |
| `/api/downloads/status` | GET | SSE stream for queue changes |
| `/api/downloads/:id/retry` | POST | Retry failed download |
| `/api/downloads/:id/clear` | POST | Remove item |
| `/api/downloads/clear-completed` | POST | Remove all completed/failed |

## Key Details
- **One download at a time**: Queue processes sequentially
- **SHA-256 verification**: Downloaded file hashed and compared against expected
- **Auto-move**: After download, file moved to `data/downloads/{version_id}/{Games|DLCs|Updates}/{filename}`
- **Game completion**: When all files for a game are done, `game_state.available = 1`
- **Retry**: Up to 3 retries before marking as failed
- **120s timeout**: Fetch uses AbortSignal.timeout(120000)
- **pkg_url**: Stored directly from TSV's PKG direct link column
