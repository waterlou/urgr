## Critical Bugs Fixed

1. **`libs/rom-manager/src/db/mod.rs` `update_game_available`**: `INSERT OR REPLACE` was destroying `rating`/`favourite`/`play_count`/`updated_at`. Changed to `INSERT ... ON CONFLICT DO UPDATE SET available = excluded.available, updated_at = datetime('now')`.

2. **`libs/rom-manager/src/dat/offlinelist.rs`**: Lost `<system>` field capture during schema migration. Re-added `in_configuration`/`system` parser state. Restored `platform: system.clone()` in game finalize. Restored test assertion `assert_eq!(games[0].platform, "Nintendo - Game Boy")`. Added `platform: String` field to `ParsedGame` struct; updated mame.rs / logiqx.rs / clrmamepro.rs / db/mod.rs / builder.rs / verifier.rs fixtures to set `platform: String::new()`. Wired `game.platform` into `insert_game` SQL with `CASE WHEN excluded.platform != '' THEN excluded.platform ELSE platform END` so first-insert wins, empty updates don't clobber existing values.

3. **`apps/rom-manager-ui/server/routes/collections.js`**: `versions_tags` SQL used `':'` as inner separator, but `parseVersions` in games.js used `'||'`. Made collections.js use `||` to match. Both files now consistent: outer `,`, inner `||`.

4. **`apps/parse-cli/src/main.rs`**: Re-enabled `--platform` flag — was declared but unused. Now overrides `game.platform` before insert. Removed duplicated `// Skip games that don't exist yet` comment (cosmetic).

5. **`libs/rom-scraper/src/sources/arcadedb.rs` `urlencoding`**: `c as u8` truncated multi-byte UTF-8 (MAME games with Japanese names like `三国志`). Rewrote with `c.encode_utf8(&mut [0u8; 4])` to percent-encode full UTF-8 bytes. Added 2 unit tests.

6. **`libs/rom-scraper/src/sources/libretro_thumbnails.rs` `url_encode`**: Same UTF-8 bug + inefficient `c.to_string().bytes()`. Same fix. Added 2 unit tests.
