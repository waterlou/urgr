# Session: 2026-05-25 (platform-based ROM organization)

## Goal
Restructure ROM output to organize by platform, add samples/CHD support, and update FBNeo/FBA import pipeline.

## Changes made

### Database & Import
- **`apps/rom-manager-ui/server/index.js`**: FBNeo/FBA import now uses `gamesByPlatform` Map. Extracts platform from DAT filename via `PLATFORM_MAP` (arcade, neogeo, nes, snes, megadriv, etc.). Inserts `game_entries` with `platform` field populated.
- **`libs/rom-manager/src/models.rs`**: Added `platform: String` to `GameEntry` struct.
- **`libs/rom-manager/src/db/schema.rs`**: Added `platform TEXT DEFAULT ''` column to `game_entries`.
- **`libs/rom-manager/src/db/mod.rs`**: Updated `list_games`, `insert_game`, `insert_games_batch` to read/write `platform` column.

### Builder output restructuring
- **`libs/rom-manager/src/builder.rs`**:
  - ROMs output → `{collection_dir}/roms/{platform}/{game}.zip` (was flat in collection_dir)
  - Samples → copies `{import_dir}/samples/*.zip` to `{collection_dir}/samples/`
  - CHD → copies `{game_name}/{disk}.chd` directories (found alongside source zips) to `{collection_dir}/chd/{game_name}/`
  - ImportIndex enhanced with `find_chd_files()` for CHD discovery
  - Cleanup walks `roms/` only; also cleans stale `chd/` directories
  - Update-mode auto-migrates old flat ROMs into `roms/` subfolder on first build

### Rust parsers / tests
- **`dat/clrmamepro.rs`**, **`logiqx.rs`**, **`mame.rs`**, **`verifier.rs`**, **`scanner.rs`**: Added `platform: String::new()` to all `GameEntry` constructions.

## Output structure
```
{base}/{source}/{version}/
  roms/                    ← game zips
    game1.zip
    arcade/game3.zip
  samples/                 ← from import_dir/samples/
    sshot.zip
  chd/                     ← CHD files from alongside source zips
    game1/
      game1.chd
  _build_status.json
```
