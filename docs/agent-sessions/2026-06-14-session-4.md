## Design

### Server endpoint: `GET /api/filesystem/browse?path=<path>`
- Returns `{ path, parent, entries: [{name, type}...] }`
- Allowed roots (hard-coded): `$HOME`, `$HOME/..`, `data/roms`, `/tmp`, `/Volumes` (macOS)
- Rejects paths outside allowed roots with 403 + `allowed_roots` list
- Resolves `~`, relative paths (→ $HOME), and `../` traversal before checking
- Directories sorted first, then files, both alphabetically; hidden files (`.name`) omitted
- Parent is null when above an allowed root (stops navigation at safe boundary)

### DirectoryBrowserModal
- MUI `Dialog` with breadcrumb path, "↑ up" button, scrollable entry list
- Single-click folder to navigate in; files shown greyed (not clickable)
- "Select this folder" returns current path to parent
- Loading spinner, error alert with fallback to home on access denied

### BuildManager integration
- "Browse..." outline button next to Import Directory text field
- Modal opens with current `buildImportDir` as starting path
- Select populates the text field; user can still type paths manually
- No changes to the build/scan logic — the text field value is used as before
