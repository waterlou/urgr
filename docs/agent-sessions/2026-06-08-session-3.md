# Session: 2026-06-08 (Part 3 - Scan/Build Unification)

## Goal
Unify scan and build across all collection types (NPS, DAT), fix routing, add download manager auto-path, and document CLI specs.

## Files Changed

### Backend
- **`server/routes/collections.js`**: Unified `POST /api/collections/:id/build` endpoint:
  - **scan**: Always uses `collectionDir` (`data/roms/{collection_folder}`), no `import_dir` needed. Routes to `nps-cli scan` or `build-cli scan` based on source. CLI updates `scanned_games`, server updates `game_state.available`. Returns `{ exists, reused, missing }`.
  - **build + NPS**: Routes to `nps-cli build`
  - **build + DAT**: Routes to `build-cli build` with progress streaming
  - `import_dir` required only for DAT builds (not scans or NPS)
- **`server/downloader.js`**: Fixed destination path to use `data/roms/{collection_folder}/{platform}/{Games|DLCs|Updates}/`
- **`server/db.js`**: Added `reloadDb()` function to reload from file after CLI modifications, `pkg_url` column to `rom_entries`, `download_queue` table
- **`server/cli.js`**: Added `nps` to `CLI_NAMES`, `needsDb`, and `findBinary` env key
- **`server/nps.js`**: Store `pkg_url` in `rom_entries` during import, use exact PKG filename from TSV URL
- **`server/routes/games.js`**: Return `available` from `game_state` in game detail

### Frontend
- **`src/components/BuildManager.jsx`**: Scan result shows `exists`/`reused`/`missing`, hides `reused` when 0
- **`src/components/GameDetail.jsx`**: Download button moved after ROM Files section

- **`src/App.jsx`**: Downloads view routing, queue count state
- **`src/api.js`**: Download API functions + SSE subscription

### Rust
- **`apps/nps-cli/src/main.rs`**: `cmd_scan` now updates `scanned_games` instead of `game_state`. Output matches `build-cli` format: `{ total_files, matched_games, missing_games }`. Removed `reset_all_unavailable` / `update_game_available` calls.
- **`libs/rom-manager/src/db/schema.rs`**: Added `region TEXT`, `UNIQUE(version_id, name, region)`
- **`libs/rom-manager/src/models.rs`**: Added `region: Option<String>` to `GameEntry`
- All DAT parsers: Added `region: None` to `GameEntry` creation

### Docs
- **`AGENTS.md`**: Added CLI specifications section describing `scan`/`build` commands, versioned vs unversioned behavior

## Key Design Decisions
- **Scan never needs import_dir**: Always uses `data/roms/{collection_folder}`
- **CLIs write to `scanned_games`, server reads and updates `game_state`**: Both `nps-cli` and `build-cli` follow this pattern
- **`reused` is versioned-only**: Only meaningful for FBNeo/MAME with cross-version build history
- **`reused = 0` for unversioned**: NPS, No-Intro have no concept of version reuse
- **Frontend hides `reused` when 0**: Clean display for unversioned collections
