### Server

### Docs
- `docs/version-fallback.md` — Updated Scanner Directory Rule to describe the `.version`-based version chain scan.

## Key Decisions
- Build completion scan must read `.version`, walk versions in order (oldest first), and stop at the built version. Never scan root recursively.
- Progress reporting uses adaptive interval `(need_copy.len() / 100).max(1)` for ~100 evenly-spaced updates.
- `processed_count` tracks ALL games processed (including missing), so progress smoothly advances even when most ROMs are missing.
